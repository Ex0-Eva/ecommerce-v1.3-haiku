# Database Installation Guide – Vercel Postgres & Prisma (PLANNED)

## Overview
This guide documents the **planned migration** from Supabase to **Vercel Postgres** with **Prisma ORM**, enabling greater flexibility and reducing vendor lock-in.

---

## 1️⃣ Prerequisites
- **Vercel account** with the project already linked (`ecommerce‑v1.3‑haiku`).
- **pnpm** installed (project uses pnpm, not npm or yarn).
- Node.js ≥ 18.
- Access to the repository at `C:\Users\User\Documents\GitHub\NeuralLink\ecommerce‑v1.3‑haiku`.

---

## 2️⃣ Create Vercel Postgres Database
1. Open the Vercel dashboard → **Postgres** → **New Database**.
2. Choose the same project (`ecommerce‑v1.3‑haiku`) and a region close to your users (e.g., `iad`).
3. Click **Create**. Vercel will provision the DB in seconds.
4. After creation you will see an environment variable named **`VERCEL_PG_URL`**. This contains the full connection string.

---

## 3️⃣ Configure Local Development
1. Open (or create) `.env.local` at the project root.
2. Add the following line (Vercel injects it automatically in the cloud, we mirror it locally for dev):
   ```dotenv
   DATABASE_URL=${VERCEL_PG_URL}
   ```
3. Keep all existing keys (e.g., `NEXTAUTH_SECRET`, `NEXTAUTH_URL`, `NEXT_PUBLIC_SUPABASE_URL`, etc.) – they are unrelated to the new DB.
4. **IMPORTANT:** The Supabase `SUPABASE_SERVICE_ROLE_KEY` is no longer required and can be removed.

---

## 4️⃣ Add Prisma to the Project
```bash
# From the project root
pnpm add prisma @prisma/client @next-auth/prisma-adapter
# Initialise Prisma (creates prisma/ folder and schema.prisma)
npx prisma init --datasource-provider postgresql
```

### 4.1 Update `prisma/schema.prisma`
Replace the autogenerated datasource block with:
```prisma
 datasource db {
   provider = "postgresql"
   url      = env("DATABASE_URL")
 }
```

### 4.2 Translate Supabase schema to Prisma models
Copy the table definitions from `supabase-schema-v12.sql` into Prisma models. A minimal example is provided below – extend it with any custom tables you have.
```prisma
model User {
  id            String   @id @default(cuid())
  name          String?
  email         String?  @unique
  emailVerified DateTime?
  image         String?
  // Relations for NextAuth
  accounts      Account[]
  sessions      Session[]
}

model Account {
  id                 String  @id @default(cuid())
  userId             String
  type               String
  provider           String
  providerAccountId  String
  refresh_token      String?
  access_token       String?
  expires_at         Int?
  token_type         String?
  scope              String?
  id_token           String?
  session_state      String?
  user               User    @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model Session {
  id           String   @id @default(cuid())
  userId       String
  expires      DateTime
  sessionToken String   @unique
  user         User     @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model VerificationToken {
  identifier String
  token      String   @unique
  expires    DateTime
  @@unique([identifier, token])
}

// Example product model (adjust fields to match your Supabase table)
model Product {
  id          Int      @id @default(autoincrement())
  name        String
  description String?
  price       Decimal  @db.Decimal(10, 2)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

// Add other models: Order, OrderItem, LicenseKey, etc.
```

### 4.3 Run the migration
```bash
npx prisma migrate dev --name init
# Follow the prompts – Prisma will create the tables in Vercel Postgres.
```
If the DB already contains tables (e.g., you imported the Supabase dump), you can run:
```bash
npx prisma db push   # sync Prisma schema without a migration history (useful for first‑time sync)
```

## 5️⃣ Replace Supabase Client Calls with Prisma
1. **Delete** `lib/supabaseClient.ts` (or keep it for reference only).
2. Create a Prisma client wrapper:
   ```ts
   // lib/prismaClient.ts
   import { PrismaClient } from '@prisma/client';
   export const prisma = new PrismaClient();
   ```
3. Update API routes (e.g., `app/api/admin/products/[id]/route.ts`) to use the Prisma client. Example DELETE route:
   ```ts
   import { prisma } from '@/lib/prismaClient';
   import { getServerSession } from 'next-auth';
   import { authOptions } from '@/lib/auth';

   export async function DELETE(request: Request, { params }: { params: { id: string } }) {
     const session = await getServerSession(authOptions);
     if (!session?.user?.role?.includes('admin')) {
       return new Response('⚠️ Unauthorized', { status: 403 });
     }
     await prisma.product.delete({ where: { id: Number(params.id) } });
     return new Response(null, { status: 204 });
   }
   ```
4. Apply similar replacements for CREATE, READ, UPDATE in other admin routes and for the cart sync (`app/api/cart/sync/route.ts`).

## 7️⃣ Deploy to Vercel
1. Commit all changes (`git add . && git commit -m "Migrate to Vercel Postgres + Prisma"`).
2. Push to the repository; Vercel will automatically run `pnpm install && pnpm build`.
3. Because `VERCEL_PG_URL` is already set in the Vercel project, **no additional configuration** is needed – the app will connect to the new Postgres instance on first request.
4. Test the live site (`https://<your‑project>.vercel.app`).

## 8️⃣ Optional: Seed Initial Data
You may want to pre‑populate some reference tables (e.g., product templates). Create a script:
```ts
// prisma/seed.ts
import { prisma } from '@/lib/prismaClient';

async function main() {
  await prisma.product.createMany({
    data: [
      { name: 'Starter Pack', price: 9.99 },
      { name: 'Premium Pack', price: 19.99 },
    ],
    skipDuplicates: true,
  });
}

main()
  .then(() => console.log('🌱 Seed complete'))
  .catch(e => console.error(e))
  .finally(() => process.exit());
```
Run with:
```bash
npx ts-node prisma/seed.ts
```
You can also add this as a **Post‑install** script in `package.json` if you want it to run automatically in a fresh environment.
